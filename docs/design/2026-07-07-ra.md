# Phase 7 (Celestial 5): Ra + Glyph Alchemy

Ra is the fifth Celestial: a "pet" progression layer (four Celestial *Memories*
that gain **memory chunks** from real time inside Ra's Reality and level up),
plus the **Glyph Alchemy** lab (21 resources fed by *refining* Glyphs, combined
by reactions). Ra's levels unlock 28 rewards, many of which retro-activate
effects that earlier Celestials deferred to Ra.

Original source: `../antimatter-dimensions/src/core/celestials/ra/{ra,alchemy}.js`
+ `src/core/secret-formula/celestials/{ra,alchemy}.js`, the `Ra.memoryTick`
call in `game.js::realTimeMechanics`, `Ra.applyAlchemyReactions` in `reality.js`,
and the refinement path in `glyphs/glyph-purge-handler.js`.

## 0. Unlock & state

**Unlock.** `Ra.isUnlocked` = `V.spaceTheorems >= 36` — which is the existing
`VUnlock.raUnlock` bit (V's 36-ST reward). That reward was already wired in the
V port; Ra's tab/features now light up behind it, and `Ra.checkForUnlocks` is
gated on the same `VUnlocks.raUnlock.canBeApplied`.

**State** (`RaState` on `CelestialsState`, `player.celestials.ra`):
- `pets: [RaPet; 4]` — teresa/effarig/enslaved/v, each `{ level, memories,
  memory_chunks, memory_upgrades, chunk_upgrades }` (level starts at 1).
- `unlock_bits: u32` — the 28 Ra unlocks (ids 0–27).
- `run: bool` — Ra's Reality active.
- `charged: u16` — bitset of Infinity-Upgrade save-ids currently *charged*.
- `dis_charge: bool` — pending discharge of charged upgrades on next reality.
- `peak_gamespeed: f64` — peak game-speed factor this Reality.
- `pet_with_remembrance: i8` — pet index with Remembrance active (−1 = none).
- `momentum_time: f64` (ms) — real time since Alchemy's `momentum` unlocked.
- `alchemy: [AlchemyResource; 21]` — `{ amount, reaction }` per resource.
- `highest_refinement_value: [f64; 6]` — per base resource (power/infinity/time/
  replication/dilation/effarig), the cap ratchet.

## 1. Pets & Memories

Each pet has a **raw memory-chunk/second** rate (per `secret-formula`):
- teresa: `4·(log10(EP)/1e4)^3`
- effarig: `4·(Effarig.shardsGained)^0.1`
- enslaved: `4·(log10(timeShards)/3e5)^2`
- v: `4·(log10(infinityPower)/1e7)^1.5`

`memoryChunksPerSecond = raw · 1.5^chunkUpgrades · continuousTTBoost.memoryChunks
· GlyphSacrifice.reality · remembrance`, and is 0 unless the pet is unlocked and
Ra is running. `GlyphSacrifice.reality` (Reality-glyph sacrifice power) is 1 in
frontier (no Reality glyph type yet).

`productionPerMemoryChunk = continuousTTBoost.memories · Achievement(168) ·
Π pet.memoryProductionMultiplier`. Each pet's `memoryProductionMultiplier` is its
XP unlock's effect (`teresaXP`/`effarigXP`/`enslavedXP`/`vXP`), 1 until unlocked.

Per tick (`memoryTick(realDiff, generateChunks)`, called from the real-time
mechanics; `generateChunks` is false while storing real time): chunks accrue at
`memoryChunksPerSecond`, then memories accrue from `(chunks + newChunks/2) ·
productionPerMemoryChunk · 1.3^memoryUpgrades` (the half-chunk term makes long
ticks behave). `requiredMemoriesForLevel(l)` (l→l+1) `= floor(adj^5.52 ·
1.5^max(0,l−15) · 1e6)`, `adj = l + l²/10`; `∞` at the cap. `levelCap = 25`.

Per-pet upgrades: memory (`×1.3` memories, cost `1000·5^n`) and chunk (`×1.5`
chunks, cost `5000·25^n`), both capped at `0.5·requiredMemoriesForLevel(24)`.
Level-up spends `requiredMemories` and calls `checkForUnlocks`.

**Remembrance** (`totalPetLevel ≥ 20`): the chosen pet's chunk rate ×5, the
others ×0.5. `pet_with_remembrance` selects it.

## 2. Ra unlocks (28)

Data table `RA_UNLOCKS[(id, pet, level)]`. On level-up, any unlock with
`pet.level ≥ level` flips its bit. Grouped by what they wire into:

- **Internal to Ra:** `teresaXP`(2,5) `1+√(log10(RM)/100)`, `effarigXP`(9,5)
  `1+bestGlyphLevel/7000`, `enslavedXP`(16,5) `1+log10(totalTimePlayed)/200`,
  `vXP`(23,5) `1+totalPetLevel/50`; `effarigUnlock`(4,8)/`enslavedUnlock`(11,8)/
  `vUnlock`(18,8) gate the next pet; `chargedInfinityUpgrades`(1,2) →
  `min(12, ⌊teresa.level/2⌋)` charges; `alteredGlyphs`(3,10)/`unlockGlyphAlchemy`
  (8, effarig 2) toggle the Alteration/Alchemy systems; `perkShopIncrease`(5,15)
  raises Teresa's Perk-Shop caps; `glyphEffectCount`(10, effarig 10) 4 effects
  always; `unlockHardV`(24, v 6) flips V + a Triad study every 6 levels.
- **Wired into built systems:** `continuousTTBoost`(25, v 10) — the big one:
  `theoremBoostFactor = min(10, max(0, log10(TT)−350)/50)`, powering infinities
  (`10^15b`), eternities (`10^2b`), replicanti (`10^20b`), dilated time (`10^3b`),
  TT-gen (`10^5b`), memories/chunks (`1+b/50`), auto-prestige (`1+2.4b`).
  `achievementTTMult`(26, v 15) applies `Achievements.power` to TT-gen;
  `achievementPower`(27, v 25) raises the achievement multiplier `^1.5`.
  `improvedStoredTime`(15, enslaved 2) amplifies Enslaved stored game time
  (`20^min(level,25)`) and raises the real-time cap. `peakGamespeedDT`(19,
  enslaved 15) DT boost from peak game speed. `unlockDilationStartingTP`(6,
  teresa 25) + `autoTP`(0, teresa 1) reshape Dilation TP gain. `relicShardGlyph
  LevelBoost`(12, effarig 15) + `maxGlyphRarityAndShardSacrificeBoost`(13,
  effarig 25) + `extraGlyphChoices…`(7, effarig 1) feed glyph generation.
- **Deferred (documented cuts):** `blackHolePowerAutobuyers`(14),
  `autoPulseTime`(17), `allGamespeedGlyphs`(20), `instantECAndRealityUpgrade
  Autobuyers`(21), `autoUnlockDilation`(22) — all QoL/automation on systems not
  worth the wiring in-frontier; the *bit* is stored + displayed, the effect is
  neutral or documented.

## 3. Charged Infinity Upgrades

`totalCharges = min(12, ⌊teresa.level/2⌋)`. A charged Infinity Upgrade's effect
is replaced by a *stronger* charged variant (the "chargedInfinityUpgrades"
mechanic). We model the charged-set bitmask + the count gate; the six charged
*effect* variants are wired where each Infinity Upgrade is read (behind
`is_charged(id)`), matching the original's `chargedEffect`. Charging/discharging
happens in the Ra tab (spends a charge slot).

## 4. Glyph Alchemy

21 resources (`ALCHEMY_RESOURCE`), five tiers. Base resources (power, infinity,
time, replication, dilation, effarig) are **refined** from Glyphs; the rest are
**reaction products** of reagents. Each resource has an `effect(amount)` and a
`cap`:
- base cap `= min(25000, highestRefinementValue[name])`;
- advanced cap `= min(reagent caps)`.

**Refinement** (`glyph-purge-handler`): refining a generated Glyph adds
`0.05 · level³/1e8 · rarity/100` to its type's resource (clamped to the effective
cap, which ratchets `highestRefinementValue`). `decoherence` spills a fraction to
the other base resources. Wired into the Glyph sacrifice path: with
`unlockGlyphAlchemy` owned and "refine" selected, a sacrifice refines instead.

**Reactions** (`applyAlchemyReactions`, per Reality): for each active product,
`reactionYield` (limiting reagent, with a >100% "forcing" burst), `actualYield`
(don't drop a reagent below the product), then consume reagents and add
`baseProduction · efficiency` (efficiency = `synergism`, `reality` uses 1).
Sorted by descending `priority`. `unpredictability` gives a Poisson chance of
re-triggering — we port the mean (no RNG divergence risk since it only speeds
fill). Gated on `effarigUnlock`.

**Effects wired:** power/infinity/time → AD/ID/TD `^` power; replication →
replicanti speed ×; dilation → DT ×; effarig → relic-shard ×; dimensionality →
all-dimension ×; inflation → AD-mult `^1.05` above threshold; momentum → all-dim
power (grows with `momentum_time`); synergism → reaction efficiency; exponential
→ IP × replicanti^p; force → AD × RM^p; uncountability → passive realities/PP/s;
cardinality → replicanti-slowdown; eternity → eternity-gen power; alternation →
TG strength; decoherence → refine spill; boundless → tesseract strength (inert,
no tesseract effect); multiversal → reality amplification (inert; amplification
is deferred); unpredictability → double-reaction chance. `reality` resource →
consumed to make a Reality Glyph (**deferred**, needs the Reality glyph type).

## 5. Ra's Reality & momentum

`initializeRun` clears the other runs and sets `run`. Ra's Reality has no special
production nerf (the pets gain from real time). `peak_gamespeed` tracks the
per-Reality peak game-speed factor (for `peakGamespeedDT`), reset each Reality.
`momentum_time` accrues real time (× Achievement 175, which we don't track → ×1)
once `momentum` is unlocked; `momentumValue = min(1 + 0.005·hours,
momentum.effect)`.

## 6. Glyph Alteration

`alteredGlyphs` (Ra teresa 10) unlocks it. Three thresholds on a type's total
sacrifice power: addition `1e36`, empowerment `1e43`, boosting `1e60`
(`sacrificeBoost = log10(clamp(power/1e60,1))/2`). These add/empower/boost the
generated glyph effects. We port the thresholds + `is_added/empowered/boosted`
and `sacrifice_boost`, applied where the altered effects read; the cosmetic
colouring is cut.

## 7. Save / load

`RaState` maps to `player.celestials.ra` in the DTO (pets array, unlockBits, run,
charged set → id array, disCharge, peakGamespeed, petWithRemembrance,
momentumTime, alchemy array, highestRefinementValue object). `#[serde(default)]`
throughout so pre-Ra saves load. Round-trips real saves.

## 8. UI

A **Ra** subtab under Celestials: the four pet panels (level, memory/chunk bars,
the two upgrade buttons, remembrance toggle), the 28-unlock milestone list, the
run button, and the charged-upgrade counter. A separate **Alchemy** view (the
original puts it under Reality) shows the 21 resources with amounts/caps/effects
and reaction toggles. The animated node/flow graph is cut for a plain grid.
