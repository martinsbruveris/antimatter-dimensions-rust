# Celestials 1–4 — Teresa, Effarig, Enslaved, V

## Summary
Ported the first four Phase-7 Celestials end-to-end (engine + save/load + Vue
tab), building the shared celestial-run machinery and one module per celestial.
Each landed as its own commit on the `celestials` branch.

## What shipped
- **Shared** (`celestials/mod.rs`): `CelestialsState` on `GameState`
  (`player.celestials`), the mutually-exclusive `run` flags,
  `is_in_celestial_reality`, `clear_celestial_runs` (from
  `reality_reset_internal`), `start_celestial_reality` (a reward-free Reality
  that sets one run flag), and `celestial_reality_completion_hooks` (from
  `finish_process_reality`). DTO + encode round-trip all four blocks; Ra/Laitela/
  Pelle survive via the encode template.
- **Teresa (7.1):** pour-RM pool → `rmMultiplier` into `uncapped_rm`, the 6
  threshold unlocks (startEU grants the 6 EUs on reset), Teresa's Reality (IP/EP
  `^0.55`, glyph-TT-gen off, `runRewardMultiplier` glyph-sacrifice bonus, passive
  epGen), 4-entry Perk Shop.
- **Effarig (7.2):** Relic Shards + 4 shard unlocks, the 3-stage Reality with
  prestige-hook stage unlocks + reward-free exits, dilation-like nerfs
  (`effarig_multiplier` AD mult, `effarig_tickspeed`, glyph-level cap via the new
  `adjusted_glyph_level`, game-speed nerf), Infinity-stage IP handling
  (`totalIPMult`→1 + base cap 1e200).
- **Enslaved (7.3):** game-time storage (`enslaved_apply_time_flow` wraps the
  tick's speed factor) + release burst, the two stored-time unlocks (softcap
  threaded through `free_tickspeed_from_shards`; run, glyph-gated), and the run
  restrictions (glyph min, always-dilated AD, 8th-AD/ID/TD caps, TS192 lock,
  disabled Black Hole, TP/DT `tachyonNerf`, `^0.55` discharge nerf).
- **V (7.4):** the six main-unlock conditions + `v_unlock_celestial`, run
  modifiers (AD/EP/IP/DT `^0.5`, squared Replicanti interval), the 9
  V-achievements (`v_check_for_unlocks` runs `tryComplete` each tick), Space
  Theorems, and the `adPow` AD power.
- **GUI:** a Celestials tab (id 9) with Teresa/Effarig/Enslaved/V subtabs
  (vendored CSS), plus the snapshot views + commands for each.

## Decisions & why
- **Entry model.** A celestial reality is `reset_reality()` (reward-free) then
  set one run flag — exactly the original's `beginProcessReality(props(true))` +
  `initializeRun`. Exit is any Reality (its reset clears the flag). The
  completion hooks (Teresa best AM, Effarig stage, Enslaved completion) run only
  on the *rewarded* path, matching `giveRealityRewards`.
- **Run-modifier seam.** The AD final-multiplier transforms (Effarig compress /
  V sqrt / Enslaved always-dilated) slot into the same
  `dimension_final_multiplier` spot the existing Time-Dilation transform uses,
  in the original's order. IP/EP `^0.55`/`^0.5` go at the gain formulas; DT sqrt/
  nerf at `dilation_gain_per_second`.
- **`adjusted_glyph_level`.** Added one helper (glyphs.rs) that both Effarig's
  cap and Enslaved's minimum flow through, and swapped `glyph_effect_values` to
  use it — one seam for both celestials' glyph-level rules.
- **Availability gating.** The original gates Teresa on Achievement 147, which we
  don't wire; we gate the tab on `reality_unlocked()` and derive the rest from
  tracked records (design doc §5). Effarig unlocks via Teresa's `effarig`
  threshold; Enslaved via Effarig's Eternity stage; V via its six conditions.

## Deviations from the design doc
- The doc scoped Effarig's persistent glyph/replicanti rewards as *deferred*; the
  implementation confirms that cut (Effarig glyph type, `maxRarityBoost`,
  Infinity-stage Replicanti-cap multiplier + `bonusRG`) — `bonusRG` is 0 without
  the cap multiplier, so they're coupled and deferred together.
- EC1's Enslaved goal-1000 is deferred (not in the doc's cut list): our EC
  completion counts are `u8`/`[u8;12]`, and 1000 needs a `u16` widening across
  state/DTO/EC logic — disproportionate for one niche end-state. Noted in
  `enslaved.rs` + ARCHITECTURE.

## Surprises & gotchas
- **Teresa pour rate is in seconds.** `Teresa.pourRM` accumulates `timePoured`
  from `(now-time)/1000` (seconds); porting the ms delta directly would have
  over-poured by 1e6×. `teresa_pour_rm` converts ms→s, and a `teresa_stop_pouring`
  command zeroes the timer on button release (the original resets it when not
  pouring).
- **V's `tryComplete` hard-achievement `continue` is a latent infinite loop** in
  the original (a hard achievement whose record meets its goal while not flipped
  spins forever). We `return` instead of `continue`, which is behaviourally
  identical in frontier (hard achievements never complete without Ra's flip).
- **Effarig also nerfs game speed** (`getGameSpeedupFactor` NERFS block), not just
  AD mult/tickspeed — easy to miss; wired into `game_speed_factor`.
- **`player.celestials.teresa.timePoured` is not saved** (a runtime accumulator);
  we mirror that (not in the DTO), and V's `runRecords` are plain numbers, not
  Decimal strings.

## Follow-ups
- Effarig glyph type + `maxRarityBoost` + Replicanti-cap/`bonusRG` (glyph/
  replicanti subsystem extensions).
- Enslaved real-time storage + Reality amplification (`boostReality`), auto-
  release/auto-store, Tesseracts' cap effect, EC1 goal-1000 (u16 widening).
- V Perk-Point goal reduction; the fastAutoEC/autoAutoClean/achievementBH/raUnlock
  reward *effects* (their systems are unbuilt or Ra-gated).
- All of the above unblock (or are unblocked by) Ra (7.5), the next celestial.
