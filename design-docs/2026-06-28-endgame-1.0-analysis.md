**Headline:** v1.0 roughly **doubles the size of the expansion**. Where the old Endgame mod added
~11,100 core JS lines over the base game (+18%), v1.0 adds **~22,000 core JS lines (+35%)** — about
**+11,000 lines on top of the already-large old Endgame**, plus ~54 more Vue components than the old
mod. This is no longer "1.5 celestials worth" of content; it's a second expansion stacked on the first.

## 3. What v1.0 Adds On Top of the Old Endgame

Everything documented in the prior analysis (Endgame prestige layer, Celestial Points / Doomed
Particles, Celestial Dimensions, Celestial Matter, Endgame Masteries/Skills/Upgrades/Milestones,
Break Eternity Upgrades, Expansion Packs, Galactic Power, Ethereal, Alpha stub, Pelle Destruction
Upgrades) **still exists**. Several of those got substantially fleshed out, and a whole second tier
of systems was added. New core/secret-formula files that did **not** exist in the old analysis:

### 3.1 Divine Dimensions + Divinity (new prestige sub-layer)
- `dimensions/divine-dimension.js` (385), `divinity-upgrades.js` (75),
  `secret-formula/celestials/divinity-upgrades.js`, `divinity-milestones.js`.
- A **5th dimension type** (after Antimatter / Infinity / Time / Celestial), 8 tiers, `×7` base
  power per tier, steep cost multipliers (`1e3 … 1e45`). Produces **Divine Matter / Divine Energy**.
- Backed by **Divinity Milestones** (`firstDivine`, `divineDimensions`, `celestialSurge`,
  `finalRebirth`, `pelleQoL`, `hadronEmpowerment`, …) that gate and multiply many other systems.
- New currencies: `divineMatter`, `divineEnergy`, `divineStars`, `divinities`.

### 3.2 Duality (new Reality-side upgrade layer)
- `duality-upgrades.js` (157) + `secret-formula/reality/duality-upgrades.js`.
- `BitPurchasableMechanicState`-based upgrades bought with a new **Dual Machines** currency
  (`Currency.dualMachines`); event-driven unlocks (`registerEvents`). Gates other systems
  (e.g. Hadrons require Duality 15/16; Penteracts require Duality 25).

### 3.3 Hypercubes (Penteracts / higher cubes)
- `hypercubes.js` (170). Extends the Teresa "tesseract/octeract" cube line into **Penteracts** and
  beyond, with double-exponential costs (`pow10(pow10(105 + …))`) paid in Eternity Points, plus a
  softcap-reduction curve tied to Octeract cube boosts.

### 3.4 Large Hadron Collider + Hadrons + Accelerators
- `large-hadron-collider.js` (385), `celestials/laitela/hadrons.js` (49),
  `secret-formula/endgame/accelerators.js` (116).
- Three Hadron types (**light / exotic / …**) within Laitela that scale Singularity gain and the
  Dark Matter cap via time-accumulation formulas; an **Accelerator** milestone system
  (`Accelerators.potency / emptiness / …`) with percentage-threshold milestones.

### 3.5 Celestial Dimension Expansion (Infinity/Eternity/Break-Infinity tiers for CDs)
- `secret-formula/endgame/celestial-dimension-expansion/` — `celestial-infinity-upgrades.js`,
  `celestial-eternity-upgrades.js`, `celestial-eternity-plus-upgrades.js`,
  `celestial-break-infinity-upgrades.js`.
- Replays the base game's Infinity → Break Infinity → Eternity progression **inside the Celestial
  Dimension layer**, with its own currencies: `celestialInfinityPoints`, `celestialInfinities`,
  `celestialEternityPoints`, `celestialEternities`. New autobuyers for celestial crunch / eternity /
  dimboost / galaxy / tickspeed / dimension back this.

### 3.6 Stars / Star Power / Ethereal Extension
- `secret-formula/endgame/stars.js` (110). Eight colored **Stars** (red, orange, yellow, …) earned
  in the Ethereal layer, each raising the **exponent** of a dimension-multiplier family (AD/ID/TD…),
  gated by Dark Matter requirements with reset thresholds. Adds `starPower`, `supernovae`,
  `condenses`, `nebulae` currencies (the changelog's "Condense / Supernova / Star Power").

### 3.7 Resurgence, Null (The Void), and misc
- `resurgence-upgrades.js` (37) + `secret-formula/celestials/resurgence-upgrades.js`.
- `secret-formula/endgame/null-upgrades.js` (226) — "The Void": `nullMatter` / `nullParticles`
  currencies.
- `elemental.js` (15) + `quotes/elemental.js`, `celestials/alpha.js` quotes — a new celestial
  ("a Celestial that finds who asked").
- `storage/endgame-migrations.js` — save-format migrations specific to the expansion.

### 3.8 Alpha — promoted from stub to full celestial
- `celestials/alpha.js` grew **77 → 1,213 lines** + `secret-formula/celestials/alpha.js`.
- The old doc flagged Alpha as "partially implemented (some code commented out)." In 1.0 it's a full
  28-stage celestial run (`isDestroyed` at stage ≥ 28), with its own records, quotes, Alpha Crystals
  (flagged WIP in the changelog), and `Endgame.resetNoReward()` integration.

### 3.9 Other fleshed-out systems
- `celestials/alpha` aside, `dimensions/celestial-dimension.js` grew **284 → 1,248** and
  `ethereal.js` **29 → 141** — the Celestial Dimension and Ethereal systems are far deeper than the
  thin versions in the old mod.
- **Achievements:** normal-achievement rows grew **144 → 184** (+40 = the "FOUR new Achievement
  rows", 8 + filler per row), each now also carrying a progression stat.
- **New autobuyers** (14 new files vs base): the six celestial-dimension-expansion autobuyers plus
  `endgame`, `galaxy-generator`, `tesseract`, `divine-dimension`, `duality-upgrade`,
  `bulk-singularity`, `music-glyph-purge`, `pelle-dilation-upgrade`.
- **Glyphs:** new rarities + two more instability layers (changelog), with refactored glyph effects.

## 5. Architecture Assessment (unchanged verdict, larger surface)

The prior doc's core finding still holds: **no new architectural paradigms.** Every new v1.0 system
reuses the established framework:

- Divine Dimensions extend `DimensionState` (same base class as every other dimension type).
- Duality / Divinity upgrades use `BitPurchasableMechanicState` + `RebuyableMechanicState`.
- LHC accelerator milestones use `GameMechanicState`; expansion packs/sets use
  `SetPurchasableMechanicState`.
- All data lives in `secret-formula/…` and is surfaced through accessor singletons.

What got worse, quantitatively:
1. **Reset/`disablePostReality` plumbing** is now threaded through far more systems — nearly every
   new getter checks `!player.disablePostReality` and a stack of `DivinityMilestone.*.isReached`
   conditions (see `hadrons.js`), so the "scattered conditionals" concern from the old doc is now
   much broader.
2. **Player state** nearly doubled in `player.js` (+772 over base), with deeply nested
   `player.endgame.*`, `player.celestials.*`, `player.dimensions.divine`, `player.reality.duality*`
   trees and several new `Set()`/array structures.
3. **Save migrations** now have a dedicated `storage/endgame-migrations.js`, implying the expansion's
   save schema is versioned independently — relevant to the Rust port's save/load work
   (cf. `2026-06-28-save-load-analysis.md`).

## 6. Porting Implications (delta vs the old plan)

The old doc estimated ~8,500 lines of additional porting effort for the Endgame mod. v1.0 roughly
**doubles that** (~22k core lines over base vs ~11k before). Concretely for the Rust port:

1. **Two more dimension types now matter.** The port's `DimensionState` abstraction must comfortably
   host Celestial *and* Divine dimensions (10 dimension families total counting AM/ID/TD).
2. **More double-exponential numbers.** Penteract costs (`pow10(pow10(105+…))`), Expansion Pack
   costs (`1e(1e150)`), and Star/Divinity formulas reinforce the need for a tetration-capable number
   type (break_eternity-style), already on the radar in
   `2026-06-21-break-eternity-representation.md`.
3. **A second "replay the early game" layer.** Celestial Dimension Expansion mirrors Infinity →
   Break Infinity → Eternity *inside* the celestial layer. A port that parameterizes the
   crunch/eternity machinery (rather than hard-coding it to antimatter) will absorb this for free;
   one that doesn't will duplicate a lot.
4. **Pervasive milestone gating.** `DivinityMilestone` / `Accelerator` / `Duality` checks are read
   inside hot production getters. The port should model these as cheap, cached boolean/effect lookups
   to avoid per-tick cost.
5. **Independent save-schema versioning** (`endgame-migrations.js`) suggests the port's save format
   should namespace expansion state and carry its own migration chain.

## 7. Bottom Line

- **The old Endgame analysis remains valid as the foundation** — every system it described still
  exists in 1.0, several now much larger (Alpha 77→1,213; Celestial Dimensions 284→1,248).
- **v1.0 is a second expansion stacked on the first**: ~+22k core JS lines over base (vs ~+11k for
  the old mod), +16 new currencies, two new dimension types, and roughly a dozen new mechanics.
- **Architecturally it's still "more of the same"** — the same `DimensionState` /
  `BitPurchasableMechanicState` / `secret-formula` patterns — so a Rust port that nails the base
  game's abstractions absorbs 1.0 as additional instances, at roughly **double** the previously
  estimated expansion effort, with the main new risks being (a) tetration-scale numbers and (b) the
  much wider spread of milestone-gated conditionals through hot code.
